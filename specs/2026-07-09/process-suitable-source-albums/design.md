# Design: Process Suitable Source Albums

> Scope reminder: this spec processes listed source album folders into `etc/3-organized-files` and writes audit artifacts under `reports/album-organization-audit/2026-07-09-source-dir-summaries/`. It does not edit `src/**`, dependencies, or `etc/1-source-files/**`.

## 1. Overview

Use a cautious dry-run-then-execute workflow for the refined suitable candidate list from the 2026-07-09 audit. The workflow relies on current `organize-files` behavior: an existing resolved destination album directory is a hard error, which prevents duplicate albums from being merged accidentally.

The executable set is destination-deduplicated before processing. Candidate execution is sequential so a successful copy creates its album folder before the next candidate runs; any unexpected collision is surfaced by the immediate dry-run and recorded as blocked.

## 2. File layout

### Modified files during this spec

```text
reports/album-organization-audit/2026-07-09-source-dir-summaries/processing-runs/<run-id>/  (new execution artifacts)
reports/album-organization-audit/2026-07-09-source-dir-summaries/album-organization-audit-2026-07-09.md  (append processing summary)
reports/album-organization-audit/2026-07-09-source-dir-summaries/source-dir-summaries-json/index.json  (append processing summary pointers)
etc/3-organized-files/**  (new organized album folders only)
```

### Files explicitly NOT modified

- `etc/1-source-files/**` read-only source input.
- `src/**` no CLI changes are part of this processing spec.
- `package.json` and `package-lock.json` no dependency changes.
- Existing report JSON summaries and dry-run artifacts remain immutable inputs.

## 3. Processing workflow

For each executable candidate in §4.1:

1. Run an immediate dry-run and save stdout/stderr/exit code:
   ```sh
   ./build/dist/index.js organize-files --source-dir "$SOURCE_DIR" --dest-dir etc/3-organized-files --format json
   ```
2. If the dry-run exits non-zero, write a blocked artifact and do not execute that candidate.
3. If the dry-run exits 0, run execution and save stdout/stderr/exit code:
   ```sh
   ./build/dist/index.js organize-files --source-dir "$SOURCE_DIR" --dest-dir etc/3-organized-files --format json --execute
   ```
4. Mark the task checkbox for that candidate immediately after success or block.
5. Continue with the next candidate; do not retry with overwrite behavior or metadata repair.

## 5. Artifact layout

Use a run-specific folder so the pass is resumable and auditable:

```text
reports/album-organization-audit/2026-07-09-source-dir-summaries/processing-runs/<run-id>/
  dry-runs/success/*.json
  dry-runs/failed/*.json
  execute/success/*.json
  execute/failed/*.json
  blocked/*.json
  processing-summary.json
  processing-summary.md
```

Each command artifact should include `sourcePath`, command argv, exit code, parsed JSON stdout when possible, raw stdout fallback, and stderr.
