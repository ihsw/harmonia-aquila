# Design: Add Web Serve Supercommand

> Scope reminder: this spec touches **only** `src/**`, `__tests__/**`,
> TypeScript/lint/test config as needed, `package.json`, and `package-lock.json`.
> No edits to `bin`, `etc`, `reports`, media folders, or generated operational
> data; no `npx`.

## 1. Overview

Use a thin-adapter plus shared-library extraction. The existing Commander files
remain responsible for parsing CLI options, translating user-input errors into
Commander errors, and writing plaintext/JSON output. All reusable behavior moves
into typed functions under `src/lib/**`, satisfying FR-9 and FR-10 while keeping
the CLI surface behaviorally stable under NFR-7.

The new `web serve` command is another thin adapter. It validates host/port,
bootstraps a NestJS application, and delegates HTTP route handling to
controllers under `src/web/**` (FR-1, FR-2, FR-3). Controllers normalize query
or body input into the same option objects the CLI adapters pass to `src/lib/**`
and return JSON rows directly (FR-5, FR-6).

Because the repository currently has `@nestjs/common` but not full NestJS
bootstrap packages, implementation should add only the minimal NestJS runtime
dependencies required to create/listen on an HTTP application. If the user
confirms they meant a different `@nextjs/common` package, stop before coding and
revise this spec (NFR-8).

## 3. Shared library extraction

Each operation gets a typed function that returns rows and throws typed errors.
The function names below are suggested; exact names may change if kept
consistent and covered by tests.

| CLI command | Shared function | Web endpoint |
| ----------- | --------------- | ------------ |
| `manage-albums summarize-source-dir` | `summarizeAlbumSourceDir(options)` | `GET /manage-albums/summarize-source-dir` |
| `manage-albums fix-tags` | `fixAlbumTags(options)` | `POST /manage-albums/fix-tags` |
| `manage-albums organize-files` | `organizeAlbumFiles(options)` | `POST /manage-albums/organize-files` |
| `manage-audiobooks validate` | `validateAudiobook(options)` | `GET /manage-audiobooks/validate` |
| `manage-audiobooks crawl` | `crawlAudiobooks(options)` | `GET /manage-audiobooks/crawl` |
| `manage-audiobooks copy-and-rename` | `copyAndRenameAudiobook(options)` | `POST /manage-audiobooks/copy-and-rename` |
| `manage-audiobooks convert-file` | `convertAudiobookFiles(options)` | `POST /manage-audiobooks/convert-file` |
| `manage-audiobooks merge` | `mergeAudiobooks(options)` | `POST /manage-audiobooks/merge` |
| `manage-audiobooks set-metadata` | `setAudiobookMetadata(options)` | `POST /manage-audiobooks/set-metadata` |

Canonical shape:

```ts
export interface OperationOptions {
  execute?: boolean
}

export async function operationName(options: OperationOptions): Promise<readonly OperationRow[]> {
  // validate, plan, optionally execute, return rows
}
```

Commander adapters catch library input errors and call `command.error(message)`.
Web controllers let the shared HTTP error mapping convert the same errors to
400 responses. Unexpected errors continue to surface as failures (FR-8).

## 4. HTTP API contract

GET endpoints use query strings for scalar options:

```text
GET /manage-albums/summarize-source-dir?dirName=/music&limit=10&ignoreNonAudioFiles=true
GET /manage-audiobooks/validate?fileName=/books/book.m4b
GET /manage-audiobooks/crawl?dirName=/books
```

POST endpoints use JSON bodies with camelCase names matching the current
Commander option object names:

```json
{
  "sourceDir": "/music/source",
  "destDir": "/music/dest",
  "execute": false
}
```

Successful responses are always the row array returned by the shared function:

```json
[
  {
    "action": "would copy",
    "destination": "Artist/Album/01 - Title.flac"
  }
]
```

Errors use one stable JSON envelope:

```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "--limit must be a non-negative integer"
}
```

## 5. Web server bootstrap

`src/commands/web/serve.ts` should parse `--host` and `--port`, then call a
bootstrap helper that returns the listening application/server so tests can
start and close it without invoking the process entrypoint.

```ts
export interface ServeWebOptions {
  host: string
  port: number
}

export async function serveWeb(options: ServeWebOptions): Promise<void> {
  const app = await createWebApp()
  await app.listen(options.port, options.host)
}
```

Keep `program.parseAsync()` only in `src/index.ts`. Do not start a server during
module import; tests must instantiate commands/controllers without binding a
port unless the test explicitly exercises bootstrap.

## 6. Test updates

### 6.1 What stays the same

- Existing CLI tests continue to instantiate Commander commands and assert
  current option parsing, dry-run rows, execution guards, and error behavior.
- Existing temporary-file helpers remain the way filesystem behavior is tested.

### 6.2 What changes

- Add direct `__tests__/lib/**` coverage for shared functions where extraction
  would otherwise be covered only through CLI adapters.
- Add `__tests__/web/**` coverage for controller request mapping and error
  mapping. Prefer controller/service unit tests over socket-bound tests except
  for one bootstrap smoke test.
- Update CLI tests only where import paths or mocks change after moving logic
  into `src/lib/**`; expected rows and error messages should remain unchanged.

### 6.3 Coverage parity table

| Area | Required disposition |
| ---- | -------------------- |
| Album summarize/organize/fix tests | Kept for CLI, supplemented with lib/web request mapping tests. |
| Audiobook validate/crawl/copy/convert/merge/set tests | Kept for CLI, supplemented with lib/web request mapping tests. |
| Output formatting tests | Kept CLI-only; web does not call `writeRows`. |
| Error handling tests | Expanded to prove library error becomes Commander error and HTTP 400. |

## 7. Migration strategy

1. Confirm dependency naming: use existing `@nestjs/common` unless the user
   clarifies `@nextjs/common` before implementation (NFR-8).
2. Establish baseline lint/build/test.
3. Extract one read-only operation first, preferably audiobook `validate`, into
   `src/lib/**` and keep its CLI tests passing.
4. Add the web module/controller pattern and route the first endpoint through
   the extracted function.
5. Repeat extraction for remaining operations, running lint after each source
   edit and focused tests after each operation group.
6. Add `web serve` registration and bootstrap smoke coverage.
7. Run full verification and scope checks.

## 8. Risk Table

| Risk | Likelihood | Mitigation |
| ---- | ---------- | ---------- |
| `@nextjs/common` was not a typo | Medium | Stop and ask before replacing the existing NestJS package family (NFR-8). |
| CLI behavior changes during extraction | Medium | Keep CLI adapter tests and compare JSON rows/error messages before and after. |
| Web routes accidentally execute writes by default | Medium | Centralize `execute` defaulting to `false` and test POST dry-runs. |
| NestJS decorators require TS config changes | Medium | Make config changes explicit and run build/lint immediately. |
| Large command files remain too large after adapters | Low | Move reusable logic into `src/lib/**`; keep new files under 200 lines. |
| Long-running media operations block requests | Medium | Preserve `p-limit` bounds and document no job queue/persistence in scope. |

## 9. Verification

After every source code file edit:

1. `npm run lint` — must exit 0 (NFR-1).

Focused checks during migration:

1. `./node_modules/.bin/vitest run __tests__/commands/manage-audiobooks/validate.test.ts`
2. `./node_modules/.bin/vitest run __tests__/commands/manage-albums/organize-files.test.ts`
3. `./node_modules/.bin/vitest run __tests__/web`

At the end:

1. `npm run lint` — must exit 0.
2. `npm run build` — must exit 0.
3. `npm test` — must exit 0.
4. `node build/dist/index.js web --help` — must list `serve`.
5. `node build/dist/index.js web serve --help` — must list `--host` and
   `--port`.
6. Start `node build/dist/index.js web serve --host 127.0.0.1 --port 0` in a
   controlled smoke test or automated test and close the server cleanly.
7. `git --no-pager diff --stat -- bin etc reports` — must show no unintended
   files.
